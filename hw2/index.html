Overview

This project explored several fundamental techniques in geometry processing and mesh editing. I implemented de Casteljau's algorithm for Bezier curves and surfaces, area weighted vertex normals for smooth shading, and mesh operations like edge flips, edge splits, and loop subdivision. This porject showed how local changes to connectivity and interpolation rules can greatly affect the final shape and smoothness of a model.

Section I: Bezier Curves and Surfaces

Part 1: Bezier curves with 1D de Casteljau subdivision

de Casteljau's algorithm works by taking a set of control points and repeatedly interpolating between neighboring points using the parameter t. Each round produces a new, smaller set of points, and after enough rounds only one point is left, which lies on the Bezier curve. I implemented this by computing one interpolation step at a time and recursively reducing the control polygon until the final evaluated point was obtained.

Part 2: Bezier surfaces with separable 1D de Casteljau

de Casteljau's algorithm extends to Bezier surfaces by applying the same repeated linear interpolation in two parameters instead of one. In my implementation, I first evaluated each row of control points in one direction using u, which produced one intermediate point per row, and then I applied the same 1D de Casteljau evaluation to those intermediate points in the other direction using v to get the final point on the surface.

Section II: Triangle Meshes and Half-Edge Data Structure

Part 3: Area-weighted vertex normals

I implemented area weighted vertex normals by iterating around all faces incident to a vertex using the half-edge structure, computing each face normal with a cross product of two edge vectors, and summing those normals together. Since the magnitude of the cross product is proportional to triangle area, larger faces automatically contribute more to the final normal. After accumulating all neighboring face normals, I normalized the result to get the final unit vertex normal used for smooth shading.

Part 4: Edge flip

I implemented edge flip by first checking whether the selected edge was on the boundary and returning immediately if it was. Otherwise, I collected the two triangles adjacent to the edge, identified their six halfedges, four vertices, and two faces, and then reassigned the halfedge connectivity so that the shared edge changed from (b,c) to (a,d) without creating or deleting any mesh elements. After rewiring the halfedges with setNeighbors, I updated the affected vertex and face halfedge pointers so that the local mesh structure stayed consistent. For debugging, the most useful trick was to think of the operation as a before and after diagram and explicitly track every halfedge, vertex, and face around the two triangles, since edge flips are easy to break if even one pointer is left inconsistent.

Part 5: Edge split

I implemented edge split by first collecting the two triangles around the selected edge and all of their existing halfedges, vertices, edges, and faces. I then created the new midpoint vertex, assigned it the average of the two endpoint positions, created the additional halfedges, edges, and faces needed for the split, and rewired all local connectivity so the original two triangles became four smaller triangles. After that, I updated the halfedge pointers for every affected vertex, edge, and face to keep the mesh consistent. For debugging, the most helpful trick was drawing the before-and-after connectivity and assigning names to every local halfedge, since edge split is easy to get wrong if even one pointer or face cycle is inconsistent.

Part 6: Loop subdivision for mesh upsampling

I implemented Loop subdivision in three main stages. First, I computed the updated positions for all original vertices and the midpoint positions for all original edges using the loop subdivision weights, storing them in newPosition without changing the mesh yet. Then I split every original edge, marked the new vertices and edges appropriately, and flipped the new edges that connected an old vertex to a new vertex so the mesh formed the correct subdivision pattern. Finally, after all connectivity changes were done, I copied the stored newPosition values back into each vertex's actual position. For debugging, the most helpful trick was separating the algorithm into clear phases and storing all new positions before modifying the mesh, since changing geometry too early would make later traversals use the wrong neighborhood information.
After Loop subdivision, the mesh becomes smoother and more rounded overall, so sharp corners and hard edges gradually soften. This is because the subdivision step updates each vertex using weighted averages of its neighbors, which pulls extreme features inward over repeated iterations. I noticed that corners that were originally very distinct started to look more like smooth curves, and flat faces became less visually rigid. Pre-splitting some edges near sharp features can reduce this effect slightly, because it gives those areas more local detail before subdivision and makes the smoothing more evenly distributed.
