Overview

This assignment builds a basic rasterization pipeline by introducing several core techniques used to solve common problems in computer graphics. Over the course of the project, I implemented methods for antialiasing, transforms, color interpolation, and texture mapping, each of which plays an important role in producing smooth and visually accurate rendered images. By working through these parts, I was able to see how individual graphics concepts fit together into a complete rendering process.

Task 1: Drawing Single-Color Triangles

For triangle rasterization, the main challenge is that pixels are discrete and finite, so a triangle cannot be represented perfectly on the screen. Instead, rasterization treats the triangle as a region that determines which pixel samples should be filled. My approach was to use the triangle's bounding box and test whether each sample point lies inside the triangle. If the sample is inside, I color that pixel sample with the triangle's color, otherwise, I leave it unchanged.

Task 2: Antialiasing by Supersampling

For supersampling, the key is dividing each pixel into subpixels and averaging the color values from those subdivisions. This is useful because it smooths out pixel colors and reduces drastic changes between pixels. Because of that, it helps reduce aliasing and makes edges look less jagged. In the rasterization pipeline, this changes the process from sampling N pixels to sampling sample_rate * N pixels, then averaging them into the final framebuffer.

Task 3: Transforms

For transforms, I implemented translation, scaling, and rotation using 3x3 transformation matrices in homogeneous coordinates. Translation moves the object, scaling changes its size, and rotation changes its orientation, and these are applied by multiplying the matrices with each vertex position.

Task 4: Barycentric coordinates

Barycentric coordinates describe a point inside a triangle using three weights, one for each vertex, and those weights always add up to 1. These weights tell us how much each vertex contributes to the point, which makes them useful for interpolating values like color. In the gradient image, each triangular section blends colors smoothly because the color at every pixel is computed as a weighted combination of the three vertex colors.

Task 5: "Pixel sampling" for texture mapping

Pixel sampling is the process of choosing what texture color a pixel should get after mapping a texture onto a triangle. I implemented this by first using barycentric interpolation to find the corresponding (u,v) texture coordinate for each sample point, then sampling the texture at that coordinate to get the final color. Nearest picks the closest texel color directly, while bilinear blends the four surrounding texels to produce a smoother result. Nearest is simpler and sharper, but bilinear looks less blocky and more natural.

Task 6: "Level Sampling" with mipmaps for texture mapping

Level sampling is the process of choosing which mipmap level of a texture to sample from based on how much the texture changes across the screen. I implemented it by computing the change in UV coordinates between nearby sample points, using that to estimate the appropriate mip level, and then sampling from that level during texture mapping. This helps reduce aliasing when the texture appears small or distorted, since using only the base level can introduce noisy or jagged results. The main tradeoff is between speed, memory, and image quality. Nearest pixel sampling is the fastest but can look blocky, while bilinear sampling is slightly slower and smoother because it blends nearby texels. For level sampling, always using level 0 is simpler and uses less work, but choosing mip levels gives better antialiasing when textures are minified, at the cost of extra memory for storing the mipmaps. Increasing the number of samples per pixel gives stronger antialiasing overall, especially along edges, but it also increases rendering time the most since more samples need to be rasterized and averaged.
